# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest on `main` | ✅ Yes |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Open a [GitHub Issue](https://github.com/mohamed-benoughidene/openreview-cli/issues)** with the title `[SECURITY] Vulnerability Report`
2. **Do NOT** include sensitive details (API keys, credentials, production data) in the issue
3. Describe the vulnerability, steps to reproduce, and potential impact

## What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix or mitigation**: Depends on severity, but we aim for fast resolution

## Scope

**In scope:**
- Code execution vulnerabilities
- Authentication/authorization flaws
- Data exposure or leakage
- Dependency vulnerabilities

**Out of scope:**
- Issues in third-party dependencies (report to them directly)
- Social engineering attacks
- Denial of service attacks
- Issues requiring physical access to a user's machine

## Disclosure Policy

We follow coordinated disclosure. Please give us reasonable time to fix the issue before making it public. We will credit reporters in the fix announcement unless they prefer to remain anonymous.
